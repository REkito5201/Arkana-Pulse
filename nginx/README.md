# Nginx в Arkana Pulse

## Что это и зачем

**Nginx** — единственная точка входа для HTTP-трафика (порт 80). Он принимает запросы снаружи и **проксирует** их на сервис `api` (FastAPI на порту 8000). Приложение не открыто в интернет напрямую.

- **Зачем:** единая точка для заголовков безопасности, лимитов, позже — TLS (HTTPS). Меньше поверхность атаки: до бэкенда доходят только «нормальные» запросы.

## Как устроено

- **`nginx.conf`** — главный конфиг: воркеры, логи, лимит размера тела запроса, зона для rate limit.
- **`conf.d/arkana.conf`** — виртуальный хост: `upstream` на `api:8000`, один `location /` с проксированием и ограничением частоты запросов.

В Docker трафик: **клиент → nginx:80 → api:8000**. Порт 8000 наружу не публикуется.

## Важные настройки (что и почему)

| Настройка | Что делает | Зачем |
|-----------|------------|--------|
| `server_tokens off` | Убирает версию Nginx из заголовков ответа | Меньше данных для разведки атакующего |
| `client_max_body_size 1m` | Запрещает тела запросов больше 1 МБ | Защита от переполнения и злоупотреблений |
| `limit_req_zone` / `limit_req` | Ограничивает число запросов с одного IP (30 r/s, burst 20) | Дополнение к rate limit в приложении (defense in depth) |
| `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy` | Заголовки безопасности в ответах | Снижение рисков XSS, clickjacking, утечки referrer |
| `proxy_set_header X-Forwarded-*` | Передаёт в приложение реальный IP и схему (http/https) | Корректные логи и проверки за прокси |
| `location /healthz` | Проксирует проверку здоровья на API | Удобно для оркестраторов (K8s, Docker) |

## Запуск

Из корня проекта:

```bash
docker compose up -d
```

Сайт доступен по адресу **http://localhost** (порт 80).

## HTTPS

- Пример конфига с TLS и редиректом HTTP→HTTPS: **`conf.d/arkana-ssl.conf.example`**. Скопируйте в `arkana-ssl.conf`, поместите сертификаты в `nginx/ssl/` (например, `fullchain.pem`, `privkey.pem`). Чтобы не было двух серверов на порту 80, переименуйте или удалите `arkana.conf` (или отключите его, переименовав в `arkana.conf.disabled`), затем перезапустите nginx.
- Самоподписанный сертификат для разработки: из корня проекта выполните `./scripts/generate-dev-cert.sh`, затем включите HTTPS-конфиг по инструкции выше.
- В production используйте Let's Encrypt (certbot) или сертификаты вашего CA.

## Логи в JSON

В `nginx.conf` задан формат **`json`** для access-лога. Чтобы писать access-лог в JSON, в нужном `server` укажите: `access_log /var/log/nginx/access.log json;`
